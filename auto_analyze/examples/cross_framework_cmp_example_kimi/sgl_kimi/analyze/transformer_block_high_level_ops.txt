NOTES ON EXECUTION FLOW AND STREAM PARALLELIZATION

1. STREAM OVERLAPS IN DECODE:
   - During CUDA graph capture (decode), Q-A norm and KV-A norm are overlapped
     on main_stream and alt_stream respectively (forward_mla.py:116-121).
   - Q-B projection runs on alt_stream during decode (forward_mla.py:194-196).
   - In forward_normal_dual_stream (MoE layers), shared experts run on main_stream
     while router+routed experts run on alt_stream (deepseek_v2.py:626-666).

2. CROSS-LAYER BOUNDARY EFFECTS:
   - RMSNorm + residual-add at layer start is fused: the residual from the
     previous layer's MLP output is added to hidden_states inside the
     RMSNorm kernel (fused_add_rmsnorm).
   - When allreduce_fusion is enabled (NOT in this run), the MLP all-reduce
     would be fused with the next layer's input layernorm. Here it's disabled.
   - The final layer (layer 60) feeds into a standalone RMSNorm before the
     LM head logits computation.

3. QUANTIZATION PATHS:
   - NVFP4 quantization applies to all linear layers in MoE experts (gate_up, down).
   - Shared experts also use FP4 quantization (same quant_config).
   - Attention projections (fused_qkv_a, q_b, kv_b, o_proj) use FP4.
   - Absorbed W_kc and W_vc BMMs use FP8 (per_tensor_quant_mla_fp8 + bmm_fp8).
   - KV cache stored in fp8_e4m3.
   - Router gate weight remains in original precision (BF16/FP32).

4. MOE SPECIFICS:
   - 384 routed experts with n_group=1, topk_group=1.
   - Each expert has intermediate_size=2048 (gate_up projects to 2*2048=4096,
     SwiGLU reduces to 2048, down projects back to 7168).
   - routed_scaling_factor=2.827 is applied to routed expert output (fused in topk
     on CUDA, since should_fuse_routed_scaling_factor_in_topk is true for trtllm).
   - Shared expert has same intermediate_size (2048*n_shared_experts=2048).
   - With TP=8 and no EP, each GPU handles 384/8=48 routed experts locally.
   - FlashInfer TRTLLM MoE kernels fuse GEMM1 (gate+up) + SwiGLU + GEMM2 (down)
     + output combine into a single kernel call.

5. MLA SPECIFICS:
   - Multi-head Latent Attention (MLA): compresses KV into a low-rank latent
     space (kv_lora_rank=512 + rope_dim=64 = 576 total KV head dim).
   - The absorbed W_kc and W_vc matrices are pre-computed and stored as
     [num_local_heads, qk_nope_head_dim, kv_lora_rank] and
     [num_local_heads, kv_lora_rank, v_head_dim] respectively.
   - In prefill (MHA_CHUNKED_KV mode), the full KV-B projection is computed
     explicitly, producing standard K and V tensors for attention.
   - In decode (MLA mode), the absorbed path avoids explicit KV-B projection
     by folding it into BMMs before/after attention.
