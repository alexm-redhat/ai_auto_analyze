================================================================================
KEY ARCHITECTURAL DIFFERENCES (affecting operation matching)
================================================================================

1. ALLREDUCE FUSION
   vLLM : Fuses AllReduce + RMSNorm + residual-add into a single kernel
          (flashinfer_trtllm allreduce_fusion_kernel_oneshot_lamport).
          The AllReduce from the PREVIOUS layer's output is absorbed into
          THIS layer's input RMSNorm. Result: NO explicit AllReduce at block
          end; the next block's input norm handles it.
   SGL  : AllReduce is SEPARATE from RMSNorm (cuMemCreate probe failed,
          fusion disabled). Uses custom 1-stage symmetric-memory AllReduce
          (cross_device_reduce_1stage) + standalone FlashInfer fused_add_rmsnorm.
          Result: EXPLICIT AllReduce at block end before next layer.

2. SHARED EXPERT TP POLICY
   vLLM : Shared expert IS TP-sharded (default TP size).
          gate_up: [7168]->[512/rank], down: [256/rank]->[7168/rank].
          Smaller per-GPU GEMMs, but output is TP-partial.
   SGL  : Shared expert uses tp_size=1 (FlashInfer backend condition).
          gate_up: [7168]->[4096], down: [2048]->[7168].
          Full-size per-GPU GEMMs, output is replicated.

3. MOE STREAM ASSIGNMENT
   vLLM : Routed MoE on MAIN stream (23), shared expert on SECONDARY stream (3649).
          Main stream is the critical path (41.5 us MoE vs 22.4 us shared).
   SGL  : Shared expert on MAIN stream (7), routed MoE on ALT stream (3488).
          Main stream is the critical path (36.5 us shared+AR vs 32.9 us MoE).

4. TORCH.COMPILE vs STANDALONE KERNELS
   vLLM : torch.compile + Inductor generates fused Triton kernels for small ops
          (e.g., Q-A norm splits into pointwise+reduction Triton kernels;
          kv_a_norm + RoPE + reshape fused into one Triton kernel).
   SGL  : Uses standalone FlashInfer/sgl-kernel CUDA kernels for norms, RoPE,
          activations. No torch.compile fusion.

5. ATTENTION Q PREPARATION
   vLLM : Explicit Triton kernel to concat q_nope_out(512) + q_pe(64) and
          apply softmax_scale before attention. (torch.compile generated)
   SGL  : TRT-LLM MLA attention kernel receives q_nope_out and q_pe as
          separate tensors; concat+scale is fused INSIDE the attention kernel.

6. ROPE + KV CACHE FUSION PATTERN
   vLLM : [kv_a_norm + RoPE + reshape] -> [concat + FP8_quant + cache_write]
   SGL  : [kv_a_norm on alt_stream] -> [RoPE + FP8_quant] -> [cache_write]

================================================================================
PHASE-LEVEL WALL TIME BREAKDOWN (estimated)
================================================================================

Attention Phase (Groups 1-12):
  vLLM: main stream compute = 60,382 ns (no overlap during attention)
  SGL:  main stream compute = 56,976 ns + 1,568 ns alt overlap = ~55,400 ns wall

  Attention wall estimate:
    vLLM: ~60,382 ns
    SGL:  ~55,400 ns (saves ~1,568 ns from kv_a_norm overlap)
    Delta: SGL faster by ~5.0 us

  Key drivers of SGL's attention advantage:
    - No fused AllReduce in input norm saves 3.7 us kernel time
      (but adds 6.7 us at block END -- net structural shift)
    - vLLM has extra Q concat+scale kernel (+1.7 us)
    - KV-A norm overlap saves SGL ~1.6 us wall time

MoE Phase (Groups 13-30):
  vLLM: main stream MoE = 41,471 ns, shared expert = 22,400 ns (fully hidden)
        MoE wall estimate: ~41,471 ns (shared expert hidden)
        But vLLM has 8.8 us extra overhead (workspace zero, buf inits, data copy)

  SGL:  main stream = 36,543 ns (shared expert + add + allreduce)
        alt stream = 32,928 ns (routed MoE, mostly hidden)
        MoE wall estimate: ~36,543 ns (routed MoE mostly hidden)

  MoE wall estimate:
    vLLM: ~41.5 us
    SGL:  ~36.5 us
    Delta: SGL faster by ~5.0 us

  Key drivers of SGL's MoE advantage:
    - vLLM fused AR+norm (7.9 us) vs SGL separate AR(8.3 us)+norm(3.2 us)=11.5 us
      => vLLM SAVES 3.6 us here, but SGL defers AllReduce...
    - vLLM has 8.8 us extra overhead kernels (workspace init, routing copy, buf inits)
    - SGL's larger shared expert GEMMs (no TP) provide longer overlap window
      that better hides the routed MoE pipeline
    - SGL's 6.7 us block-end AllReduce is offset by not needing fused AR at
      next block's input (structural shift, not net overhead)
