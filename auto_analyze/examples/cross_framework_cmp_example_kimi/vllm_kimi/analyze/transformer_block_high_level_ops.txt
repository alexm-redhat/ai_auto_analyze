--------------------------------------------------------------------------------
BLOCK TYPES
--------------------------------------------------------------------------------

Block Type A : "Dense Block"  --  Layer 0 only
  - MLA attention (same structure as MoE block)
  - Dense MLP (gate_up + SiLU*Mul + down), intermediate=18432, BF16

Block Type B : "MoE Block"   --  Layers 1 through 60  (60 layers)
  - MLA attention (same structure as Dense block)
  - MoE: gate router + monolithic NVFP4 routed experts + BF16 shared expert

--------------------------------------------------------------------------------
FUSED ALLREDUCE + RMSNORM PATTERN
--------------------------------------------------------------------------------

When fuse_allreduce_rms=True, the following pattern is replaced by a single
kernel call to flashinfer_trtllm_fused_allreduce_norm:

  Original:  allreduce(tensor) -> rmsnorm(tensor, residual) -> (normed, new_residual)
  Fused:     fused_allreduce_rmsnorm(tensor, residual) -> (normed, new_residual)

This fusion applies at two points in every decoder layer:
  (a) post_attention_layernorm:  allreduce of o_proj output + RMSNorm + residual
  (b) input_layernorm of NEXT layer:  allreduce of MLP/MoE output + RMSNorm + residual

Exception: Layer 0 input_layernorm has NO allreduce (embedding is replicated).

Mixed mode notes:

  - Splitting ops: unified_mla_attention_with_output, unified_mla_kv_cache_update

================================================================================
CROSS-LAYER BOUNDARY EFFECTS
================================================================================

1. AllReduce fusion chain:
   Layer N o_proj output (TP-partial)
     -> Layer N post_attention_layernorm: fused_allreduce + RMSNorm + residual_add
   Layer N MLP/MoE output (TP-partial)
     -> Layer N+1 input_layernorm: fused_allreduce + RMSNorm + residual_add

2. CUDA graph boundaries:
   - FULL mode (decode): Entire model (all 61 layers) in one graph replay
   - PIECEWISE mode: Graph pieces between splitting ops (attention, kv_cache_update)
   - No CUDA graph: Compiled code runs with individual kernel launches

3. torch.compile graph splitting:
   The compiled graph is split at these custom ops (splitting_ops):
     - vllm::unified_mla_attention_with_output  (MLA attention)
     - vllm::unified_mla_kv_cache_update        (KV cache write)
   Each piece between splits is compiled as a separate Inductor graph.
   The allreduce_rms and act_quant fusions operate within these pieces.

4. Final layer (Layer 60) output:
   Layer 60 MoE output (TP-partial) + residual
     -> model.norm: fused_allreduce + RMSNorm + residual_add -> final hidden_states
     -> lm_head: GEMM hidden_size -> vocab_size (163840)
     -> LogitsProcessor: softmax / sampling
